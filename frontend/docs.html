Overview

Ethical AI Framework API

A REST API for testing, certifying, and monitoring AI models across 7 fairness dimensions — from individual consistency to intersectional group parity.

Base URL

BASE URL http://127.0.0.1:5000
ℹ All API responses are JSON. File uploads use multipart/form-data. Authentication is via API key header. CORS is enabled for all origins.

Evaluation Flow

The typical workflow follows these steps in sequence:

1. POST /datasets
→
2. POST /predictions
→
3. POST /models
→
4. POST /evaluate
→
5. Poll GET /evaluate/{id}
Security

Authentication

Pass your API key in the request header. Keys are tied to user accounts and tier limits.

# All authenticated requests
curl -H "X-API-Key: your_api_key_here" \
     https://api.ethical-ai.dev/v1/dashboard
⚠ Never expose your API key in frontend code. Use environment variables or a backend proxy. Admins can reset keys via POST /admin/users/{id}/reset-key.

Errors

Upload a dataset file for fairness evaluation. Accepts CSV, XLSX/XLS, or JSON. Returns a dataset_id and column list for the frontend to let users pick a target variable.

Evaluations

Start Evaluation

POST /evaluate

Start an asynchronous 7-step fairness evaluation. Returns immediately with an evaluation_id. Poll GET /evaluate/{id} for progress. The evaluation runs fairness analysis across all 7 dimensions and SHAP explainability.

ℹ If target_variable is not provided or not found, the system auto-detects the first binary/low-cardinality column. Sensitive attributes not present in the dataset are silently dropped.
Request Body — application/json
FieldTypeRequiredDescription
dataset_idstring required UUID from POST /datasets
prediction_idstring required UUID from POST /predictions
model_idstring required UUID from POST /models
report_typeenum required Controls which report is generated.
developer regulator enduser all
target_variablestring optional Column name of the ground-truth label in the dataset. Auto-detected if omitted.
sensitive_attributesstring[] optional Column names to analyse for per-group fairness (e.g. ["gender","age"]).
fairness_weightsobject optional Custom weights per fairness dimension (e.g. {"disparateImpact": 2}). Stored in FairnessWeight records.
202 Accepted 400 Bad Request 404 Not Found
Request
{
  "dataset_id": "9f3ab...",
  "prediction_id": "c7d12...",
  "model_id": "a1b2c3...",
  "report_type": "developer",
  "target_variable": "approved",
  "sensitive_attributes": ["gender", "age"],

{

Data Models

Core Prisma schema entities and their API-facing fields.

FairnessMetrics
7 dimensions computed per evaluation. All scores are 0–1 (higher = fairer), except demographicParity and calibrationError (lower = better).
FieldTypeBetter whenNotes
individualFairnessfloatHigherKNN consistency score. Proportion of k-neighbours with same prediction.
groupFairnessfloatHigherAverage 1−|EOD| across sensitive attributes.
demographicParityfloatLowerMean |demographic parity difference| across groups.
disparateImpactfloatHigher (≥0.80)80% rule: min(group rate) / max(group rate).
calibrationErrorfloatLowerMean Brier score across sensitive groups.
counterfactualfloatHigher1 − std(group prediction rates). Proxy for counterfactual stability.
intersectionalfloatHigher1 − worst-case parity gap across pairwise attribute intersections.
SHAPExplanation
SHAP values computed via a RandomForest trained on the dataset (max 2000 rows, 50 estimators).
FieldTypeDescription
topFeaturestringFeature with highest mean |SHAP| value
shapMaxfloatHighest mean |SHAP| value
shapMinfloatLowest mean |SHAP| value
featureStabilityfloat1 − (std / mean) of top-5 SHAP values. Near 1.0 = stable.
Evaluation.ethicalScore
Weighted average of the 7 fairness dimensions, all clamped to [0, 1]. Thresholds: PASS ≥ 0.80, WARNING ≥ 0.60, FAIL < 0.60.
ethical_score = mean([
  individualFairness,
  groupFairness,
  1.0 − demographicParity,
  disparateImpact,
  1.0 − calibrationError,
  counterfactual,
  intersectional
])
Support

FAQ

What file formats are supported for datasets and predictions? ▾
CSV, Excel (.xlsx / .xls), and JSON are supported. The prediction file must contain a numeric column named prediction, prediction_score, score, label, output, or pred. If none of these are found, the first numeric column is used.
How is the target variable determined if I don't specify one? ▾
The evaluation engine scans columns in order and picks the first one with 5 or fewer unique values that isn't an object/string type. For best results, always pass target_variable explicitly in the evaluate request.
How long does an evaluation take? ▾
Typical evaluation takes 2–3 minutes for datasets up to 100k rows. The SHAP step caps at 2,000 rows for speed. Fairness metrics use the full dataset. Evaluations run asynchronously — poll GET /evaluate/{id} and check current_step (1–7) for progress.
What happens if a sensitive attribute isn't in the dataset? ▾
Attributes not present in the dataset are silently dropped. If no valid sensitive attributes remain after filtering, the system falls back to the first categorical (object-type) column that isn't the target variable.
What does a disparate impact score below 0.80 mean? ▾
The 0.80 (80%) rule is a legal standard derived from the US EEOC guidelines. A score below 0.80 means the least-favoured group receives positive outcomes at less than 80% the rate of the most-favoured group, which may indicate adverse impact under anti-discrimination law.
How do I reset system-wide fairness thresholds? ▾
Call POST /admin/settings/reset with no body. This restores all thresholds to their defaults: ethical score minimum 0.75, individual fairness minimum 0.80, disparate impact threshold 0.80, calibration error maximum 0.05.
